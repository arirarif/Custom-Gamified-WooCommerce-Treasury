Oxygen-first build WooCommerce-native product flow Plugin-first integration Phased delivery

What The Client Is Asking

  1. Build in WordPress using Oxygen Builder where possible.
  2. Work with existing installed plugins and integrate them cleanly.
  3. Implement Treasury calculations after they send conversion rules (Dz/Tz/PR/EUR).
  4. Keep product management in normal WooCommerce admin flow.
  5. Start with 3 core pages first: Shop Home, Explore, Single Product.
  6. Give realistic timeline because Mint page may be developed in parallel by another dev.
  7. Support display-only prizes that are not purchasable and link to games.
  8. Prepare architecture for next phase: Mint page + Account + game-style product cards.

Is It Possible?

Yes, fully possible. Best estimate is roughly 85-90% plugin + Oxygen configuration and 10-15% custom code for unique logic and integration glue.

  • Custom code likely needed for: Treasury conversions edge cases, display-only prize behavior, game action buttons, and cross-plugin sync safeguards.
  • Good news: WooCommerce-native product creation can remain intact while front-end behavior is customized.

What You Should Promise

  1. Oxygen-based reusable templates and component system.
  2. WooCommerce-native product operations remain standard.
  3. Plugin-first implementation, custom code only where required.
  4. Architecture that can expand into Mint and Account phases.
  5. Staged delivery so parallel development is safe.

Critical Clarifications Before Final Quote

  1. Exact plugin list + versions (BuyCred/MyCred/MarketKing/CheckoutWP).
  2. PR earning events (purchase, referral, login, game actions).
  3. Conversion formulas and rounding rules.
  4. Display-only prize behavior and visibility rules.
  5. Vendor permission matrix.
  6. Staging access, backups, and rollback expectations.

Recommended Workflow (Your Side)

  1. Discovery + technical audit of current stack.
  2. Confirm plugin responsibilities and conflict risks.
  3. Define data model for Tz, Dz, PR, and conversion math.
  4. Build Oxygen design system (buttons, cards, FAQ, utility sections).
  5. Build 3 core pages (Shop Home, Explore, Single Product).
  6. Implement behavior tiers (buyable, redeemable, display-only linked prizes).
  7. Integrate checkout redemption + points visibility.
  8. QA (mobile/desktop + plugin interactions + edge-case flows).
  9. Handoff docs + reusable component map for next pages.

Practical Time Split

  • Week 1: audit, architecture, design system, Shop Home.
  • Week 2: Explore, Single Product, display-only prize behavior.
  • Week 3: conversion logic polish, full testing, bug fixes, deployment support.

MVP for 3 pages: usually 7-10 working days if inputs are ready, plus 3-5 days for stabilization.
