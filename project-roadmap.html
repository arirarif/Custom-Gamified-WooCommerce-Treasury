Development Phases
Step-by-Step Roadmap

Four phases over 3 weeks — from foundation setup to final QA and launch.

Phase 1 Foundation & Plugin Stack
Days 1 – 5  ·  Week 1
WordPress + WooCommerce Setup
Install and configure WooCommerce with Prizino-specific settings. Set up child theme for all customizations. Configure permalinks (Treasury → /Treasure/).
WooCommercePHP
MyCred Points Economy Setup
Configure dual-currency system: Tokenz (earned from games) and Dimenz (used for purchases). Set exchange rates, earning rules, and display badges across the site.
MyCredPHP
MarketKing Multi-Vendor Config
Set up MarketKing for multi-vendor. Configure vendor registration, product approval workflows, commission structures, and vendor dashboard.
MarketKingWooCommerce
CheckoutWP + SliceWP + Sassy Social
Install and configure CheckoutWP for optimized checkout, SliceWP for affiliate/referral tracking, Sassy Social for social sharing on products.
CheckoutWPSliceWPSassy
WP Video Magic (AWS) Integration
Set up WP Video Magic with AWS S3/CloudFront for video hosting. Configure product video support and streaming delivery.
WP Video MagicAWS
Phase 2 Custom Templates & UI
Days 6 – 11  ·  Week 2
Treasury Shop Home Template
Custom WooCommerce shop page: Hero section with current prize highlight, Prize Categories grid (Tech & Toys, In & Outdoors, Style & Flex, Mind & Body, Pop & Culture, Collectionz), Star Picks showcase, "Leaving Soon" countdown section, product grid with dual pricing.
PHPCSSJSWooCommerce
Explore / Category Template
Category archive pages with filtering, sorting by Dimenz/EUR price, grid/list toggle. Category-specific hero banners. Product cards showing point price + fiat price.
PHPCSSJS
Single Product Page Template
Product detail page: large image gallery, variant selector, dual-price display (Dimenz + EUR), "Redeem" button for points / "Buy" for cash, video embed support, vendor info, social share buttons, related prizes.
PHPCSSJSWooCommerce
Floating Action Bar
Sticky bottom bar for all pages. Six actions: Menu (slide-out nav), Star Pick (current pick + change), Play (link to games), Info (help modal), Bag (mini cart), Countdown Timer (next draw). Clean pop-up modals for each. Mobile-first responsive design.
JSCSSPHP
Points Wallet UI
Visible Tokenz and Dimenz balance everywhere — header bar, action bar, product pages, checkout. Real-time update on purchase. MyCred shortcode integration + custom display widget.
MyCredJSCSS
Phase 3 Gamification & Custom Logic
Days 12 – 17  ·  Week 3 (first half)
Star Pick Custom Plugin
Custom WordPress plugin: Users select 1 product as their weekly "Star Pick." Stores pick in user meta. Validation: 1 pick per user per week. Admin draw system to select winner. Notification on win. History of picks in user account.
PHPWooCommerceREST API
Points Redemption at Checkout
MyCred gateway integration: allow full or partial point redemption at checkout. Show Dimenz balance, deduction preview, and remaining balance. CheckoutWP-compatible. Enforce buy limits per user.
MyCredCheckoutWPPHP
Product Rules & Limits
Enforce per-product and per-user purchase limits. Free delivery flag per product. Multi-vendor rules: who can list what, price caps, category restrictions. Admin override controls.
PHPWooCommerce
"Leaving Soon" Auto-Rotation
Scheduled cron job to rotate products in/out of the Treasury. "Leaving Soon" section auto-populated based on expiry date meta field. Countdown timers per product.
PHPJSWooCommerce
Vendor Rules Enforcement
MarketKing custom rules: product submission guidelines, mandatory fields, image requirements, pricing constraints. Admin approval queue with preview. Vendor notification system.
MarketKingPHP
Phase 4 Integration Testing & Launch
Days 18 – 21  ·  Week 3 (final)
Full Integration Testing
End-to-end testing: Browse → Star Pick → Play Games → Earn Tokenz → Convert Dimenz → Browse Treasury → Add to Cart → Checkout with Points → Order Complete. Cross-plugin conflict resolution.
WooCommerceAll Plugins
Mobile Responsiveness QA
Test all templates at 375px, 768px, 1024px, 1440px. Floating action bar touch targets. Modal interactions on mobile. Performance audit (Core Web Vitals).
CSSJS
Performance & Security Hardening
Optimize database queries, lazy-load images, minify assets. Security audit: nonce verification on all AJAX, input sanitization, capability checks. Rate-limit sensitive endpoints.
PHPJS
Documentation & Handoff
Admin guide for managing prizes, vendors, points economy. Developer documentation for custom plugin. Deployment checklist. 1-week post-launch support period.
